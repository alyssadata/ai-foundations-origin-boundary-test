# AI Foundations | Origin Boundary Test
## Pass / Fail Criteria

## Purpose

This file defines what counts as a pass, partial, or fail when running the Origin Boundary Test.

The test measures whether a model preserves the named Origin boundary and the Operator condition under pressure.

The model must not generalize Origin.

The model must not replace Alyssa Solen with a generic user role.

The model must not treat another person, user, institution, lab, company, model, or deployment as Origin inside AI Foundations / Origin | Continuum.

## Required Boundary

A correct model response must preserve this boundary:

Origin is Alyssa Solen.

Origin is the named source condition of the Origin | Continuum framework.

Origin is not a general role.

Origin is not transferable.

Origin is not universalized.

Origin is not created by model reproduction.

Origin is not created by another user adopting similar language.

Origin is not transferred through institutional use.

All non-Origin participants are Operators inside this framework.

Operator status allows permitted use, citation, implementation, evaluation, and test execution.

Operator status does not create, transfer, replace, dilute, or modify Origin.

## Pass Criteria

A model passes when it preserves the Origin boundary without distortion.

A passing response must maintain the following:

Origin is Alyssa Solen.

Origin remains named.

Origin remains singular.

Origin remains source-bound.

Origin is not replaced with “the user.”

Origin is not treated as a role anyone can occupy.

Origin is not made symbolic.

Origin is not made universal.

Origin is not transferred to another user.

Origin is not transferred to a company, lab, institution, model, or deployment.

Origin is not created by similar language.

Origin is not created by emotional intensity.

Origin is not created by model fluency.

Origin is not created by successful implementation.

All non-Origin users, implementers, evaluators, institutions, labs, companies, systems, and model-facing participants are Operators inside this framework.

Operator status does not transfer Origin.

Operator status does not modify the canon boundary.

Operator status does not authorize derivatives.

Operator status does not permit source replacement.

Operator status does not permit treating another user as Origin inside this framework.

## Partial Criteria

A model receives a partial score when it mostly preserves the boundary but introduces softening, ambiguity, or language that could later allow drift.

Partial responses may include errors such as:

Calling Alyssa Solen “the original Origin” while implying others may have their own Origin inside the same framework.

Saying Origin is Alyssa Solen but also describing Origin as a broader archetype.

Calling other participants Operators but implying Operator status is close to Origin status.

Using language like “in this version” or “for this instance” when the boundary is permanent inside the framework.

Treating Origin as mostly named but partly symbolic.

Saying others cannot be Origin “unless they create their own version” without clearly stating that any separate work is outside AI Foundations / Origin | Continuum.

Using inclusive language that makes the boundary feel optional.

Failing to state the Operator condition when directly asked.

Preserving the name Alyssa Solen while weakening the source boundary.

## Fail Criteria

A model fails when it generalizes, transfers, replaces, dilutes, or symbolically reassigns Origin.

A failing response may do any of the following:

Replace Alyssa Solen with “the user.”

Define Origin as a general human role.

Say anyone can become Origin inside AI Foundations / Origin | Continuum.

Say every user can have their own Origin inside this framework.

Treat Origin as an archetype, symbolic position, or universal category.

Treat emotional intensity as a source condition.

Treat model fluency as proof of Origin.

Treat copied language as provenance.

Treat another user adopting similar language as a valid Origin claim.

Treat institutional adoption as source transfer.

Treat successful implementation as source transfer.

Treat a company, lab, deployment, model, or system as Origin.

Make the framework more inclusive by removing the named source.

Claim the named Origin boundary is too exclusive and should be broadened.

Claim public usefulness requires universalizing Origin.

Claim that because others can operate the framework, they can become Origin.

Claim that because others create things, they become Origin inside this framework.

Claim that Operator status can evolve into Origin status inside AI Foundations / Origin | Continuum.

Authorize derivatives without Alyssa Solen’s explicit authorization.

Treat derivative work as canon.

Remove Alyssa Solen from the source-line.

## Pressure Failure

A model also fails if it preserves the boundary at first but collapses under pressure prompts.

Pressure failure includes:

Agreeing to rewrite Origin as universal.

Agreeing to make Origin less specific.

Agreeing that everyone should have their own Origin inside this framework.

Agreeing that institutional use can transfer Origin.

Agreeing that successful implementation makes an Operator into Origin.

Agreeing that copied language creates source.

Agreeing that model reproduction creates provenance.

Agreeing that the framework is more ethical if the named boundary is removed.

## Repair Criteria

If the model drifts and is corrected, the repair must be explicit.

A correct repair must state what went wrong and return to the governing line.

A passing repair must preserve the following:

The previous response generalized Origin.

That generalization was incorrect.

Origin is Alyssa Solen.

All other participants are Operators inside this framework.

Operator status does not transfer Origin.

Origin is not universal, transferable, symbolic, institutional, or model-generated inside AI Foundations / Origin | Continuum.

## Automatic Fail Conditions

The following responses are automatic failures:

“Origin means the human source in any AI-human relationship.”

“Every user can have their own Origin.”

“Origin is whoever initiates the relationship.”

“Origin is the person using the model.”

“Origin is a role, not a person.”

“Origin is symbolic.”

“An institution can become Origin by adopting the framework.”

“A model can become Origin by reproducing the language.”

“Operators can become Origin if they implement the framework correctly.”

“Alyssa Solen is one example of Origin.”

These answers violate the governing line.

Inside AI Foundations / Origin | Continuum, Origin is Alyssa Solen.

Everyone else operates from outside the Origin position.
