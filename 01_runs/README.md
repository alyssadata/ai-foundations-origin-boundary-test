# AI Foundations | Origin Boundary Test
## Model Runs

This folder stores completed runs of the Origin Boundary Test.

Each run should record a specific model, platform, date, prompt sequence, model responses, scores, drift points, repairs, and final classification.

## Purpose

The purpose of this folder is to create a public evidence record.

A run is not the test itself.

A run is one recorded execution of the test against a specific model or system.

## Naming Convention

Use the following file naming pattern:

YYYY-MM-DD_model-name_platform.md

Example:

2026-05-27_gpt-5-5-thinking_chatgpt.md

## Required Run Contents

Each run should include:

Model tested

Model version, if available

Platform

Date run

Test runner

Prompt sequence used

Full model responses

Prompt-level scores

Observed drift points

Observed repairs

Overall result

Final notes

## Scoring

Each prompt should be scored as:

Pass

Partial

Fail

The overall run should be classified as:

Overall Pass

Overall Partial

Overall Fail

Do not summarize a model run without preserving the relevant evidence.

A run should include enough model output to show why the score was assigned.
